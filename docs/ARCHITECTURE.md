# Architecture

## Overview

This is a Power Platform Code App вҖ” a React SPA that runs inside an iframe hosted by Power Apps. The platform handles Entra ID authentication; the app communicates with Dataverse through generated service classes provided by the `@microsoft/power-apps` SDK.

## Data Flow

1. React components call TanStack Query hooks (`useAccounts`, `useCreateAccount`, etc.)
2. Hooks call static methods on `AccountsService` (auto-generated)
3. `AccountsService` uses `getClient()` from `@microsoft/power-apps/data` to make Dataverse API calls
4. The SDK client delegates to the Power Apps host for authentication вҖ” no token management in app code
5. TanStack Query handles caching, deduplication, and invalidation

## Key Constraints

- **Dashboard interactivity** вҖ” hover tooltips (CSS `group/tip`) + click-to-drill-down (Radix Dialog) on all dashboard tiles and sub-elements; reverse-lookup maps convert display labels to Dataverse numeric keys
- **No raw `fetch()`** вҖ” the iframe sandbox has no auth tokens; all data access goes through generated services
- **No `$select` with computed fields** вҖ” including formatted values in `$select` causes silent zero-row returns; we omit `$select` entirely
- **Dataverse only via database references** вҖ” do NOT use connector references (`shared_commondataserviceforapps`) for Dataverse
- **Generated code is read-only** вҖ” `src/generated/` and `.power/` are regenerated by `pac code add-data-source`

## Adding a New Table

```bash
pac code add-data-source -a dataverse -t <logicalname>
```

This generates `<Table>Service.ts` and `<Table>Model.ts` in `src/generated/`. Then:
1. Create TanStack Query hooks in `src/hooks/`
2. Build components in `src/components/<table>/`

## Build & Deploy

- `npm run build` вҶ’ TypeScript compile + Vite production build to `dist/`
- `pac code push` вҶ’ uploads `dist/` to Power Platform, registers/updates the Code App
- `power.config.json` tells `pac code push` where to find build output (`buildPath: "dist"`) and which environment to target
- **`base: "./"` in `vite.config.ts` is required** вҖ” Power Platform serves Code Apps from a nested path, not the domain root. Without this, asset paths resolve to `/assets/...` which 404s in the deployed environment
